## 1. 打开应用

**在线使用（推荐）：** 直接访问 **https://evyd-product-roadmap.vercel.app/**

团队所有成员使用同一个地址，数据存储在云端，编辑后其他人刷新页面即可看到最新内容。

**本地开发：**

```bash
npm install            # 安装依赖
npx vercel dev         # 启动本地开发服务器（含远程存储 API）
```

> **注意：** 直接双击 `index.html` 打开也可预览界面，但无法连接远程存储，数据不会同步。

## 3. 导入数据

### 3.1 导入 CSV 文件

1. 点击工具栏「**加载 CSV**」按钮
2. 选择本地 `.csv` 文件（支持 Excel 导出的 UTF-8 CSV）
3. 数据立即渲染，并自动保存到本地
4. **导入会替换全部现有数据**，请先「保存」备份

### 3.2 下载示例文件

点击工具栏「**示例**」按钮，下载包含示例数据的 CSV 文件，可用 Excel 或文本编辑器查看格式。

### 3.3 格式要求

首行为表头，后续每行一条条目：

```csv
author,module,problem,title,description,outcome,collaborators,startMonth,duration
Lynn,数据展示,现有图表加载慢,图表组件重构,重构图表渲染层,加载速度提升50%,,4,3
Ned,用户增长,注册流失率高,注册漏斗优化,简化注册步骤,转化率提升20%,市场团队,6,2
```

| 列 | 必填 | 说明 |
|----|------|------|
| `author` | ❌ | 负责人姓名，逗号分隔多人；用于执行人筛选 |
| `module` | ✅ | 所属模块，相同名称自动归组 |
| `pillar` | ❌ | 所属 Pillar（高层价值主题）；用于 Pillar 视图；留空归入「未分配」 |
| `problem` | ❌ | 解决的问题，显示在 tooltip |
| `title` | ✅ | 条目标题，显示在条形上 |
| `description` | ❌ | 详细描述，显示在 tooltip |
| `outcome` | ❌ | 预期效果，显示在 tooltip |
| `collaborators` | ❌ | 协作团队，多个用 `;` 分隔；用于配合团队筛选 |
| `startMonth` | ✅ | 开始月份 1–12（1=一月，4=四月） |
| `duration` | ✅ | 持续月数，最长延伸至财年末（次年3月） |

## 4. 手动添加条目

1. 点击工具栏「**+ 添加条目**」按钮（或按 `⌘N` / `Ctrl+N`）
2. 在弹出的表单中填写：
   - **所属模块**（必填）：直接输入模块名，相同名称自动合并到同一行
   - **所属 Pillar**（选填）：高层价值主题，用于 Pillar 视图；留空则归入「未分配」
   - **功能/项目标题**（必填）：显示在条形上的名称
   - **负责人**（选填）：逗号分隔多人，支持执行人筛选
   - **协作团队**（选填）：逗号分隔多个团队，支持配合团队筛选
   - **解决的问题** / **预期效果** / **详细描述**（均选填）
   - **开始月份**：1–12，对应自然月
   - **持续时长**：月数
3. 表单底部的「预览」区域实时展示条形位置
4. 点击「**保存**」

> **提示：** 模块名称相同的条目会自动归到同一模块组，颜色也会一致。

## 5. 编辑与删除条目

**编辑：**
1. 单击任意条形，弹出编辑表单
2. 修改内容后点击「**保存**」

**删除：**
1. 单击条形，打开编辑表单
2. 点击左下角红色「**删除条目**」按钮
3. 在确认弹窗中点击「确认」

> **注意：** 删除操作不可撤销，如需找回请从导出的 CSV 文件恢复。

## 6. 拖拽调整

### 6.1 移动条目（调整开始月份）

- 鼠标**按住条形中间**区域并水平拖拽
- 松开后自动吸附到最近的月份
- 拖拽过程中条形半透明抬起，松手后保存

### 6.2 调整时长

- 鼠标**按住条形右侧深色手柄**（竖线区域）并水平拖拽
- 向右拖拽延长时长，向左缩短，最短 1 个月
- 时长可以**跨越日历年**，最长延伸到财年末（次年 3 月）
- 条形宽度实时更新，徽标数字同步变化

> **注意：** 拖拽后松开鼠标才算保存，期间刷新页面会丢失未保存的拖拽。

## 7. 模块管理

### 7.1 重命名模块

- **双击**左侧模块名称，进入编辑状态
- 修改名称后按 `Enter` 确认，或按 `Esc` 取消
- 该模块下所有条目的 `module` 字段同步更新

### 7.2 合并模块

- 将某个模块重命名为另一个已存在的模块名
- 两组条目自动合并到同一模块

### 7.3 拖拽排序

- 鼠标**按住模块左侧拖拽手柄**（⠿ 图标）并上下拖拽
- 松开后模块顺序即时更新并保存

## 8. Pillar 视图

Pillar 是面向**汇报与决策层**的高层分类，一个 Pillar 代表一个清晰的战略价值主题（如「用户增长」「平台稳定性」「变现能力」），可以跨越多个功能模块。

### 8.1 切换到 Pillar 视图

点击筛选栏左侧「**Pillar 视图**」标签，页面即切换为按 Pillar 分组。

- 每个 Pillar 对应一个分组行，展示该 Pillar 下所有条目
- 条形**保留模块颜色**，一眼看出每条工作来自哪个模块
- 没有填写 Pillar 的条目归入「**未分配**」分组

### 8.2 给条目指定 Pillar

**方式一：手动添加/编辑**
打开条目表单，填写「所属 Pillar」字段即可。

**方式二：CSV 批量设置**
在 CSV 中添加 `pillar` 列，填入对应的 Pillar 名称，导入后自动生效：
```csv
author,module,pillar,title,startMonth,duration
Lynn,健康管理,用户留存,减重计划,4,3
Ned,本地活动,用户留存,文莱寻宝活动,5,2
Lynn,基础架构,平台稳定性,API 网关升级,4,2
```

### 8.3 Pillar 管理

与模块管理相同：
- **重命名**：悬停 Pillar 行显示铅笔图标，点击后输入新名称，`Enter` 确认
- **合并**：将 Pillar 重命名为已有 Pillar 名称，两组自动合并
- **拖拽排序**：按住 ⠿ 图标上下拖拽调整 Pillar 顺序
- **折叠/展开**：点击 Pillar 行折叠该分组

## 9. 筛选条目

筛选栏（工具栏下方）提供两个独立筛选器，可单独或组合使用。

### 9.1 按执行人筛选

1. 点击「**执行人 ▼**」按钮，弹出负责人列表（从所有条目的 `author` 字段自动收集）
2. 勾选一个或多个执行人名称
3. Roadmap 立即只显示**至少有一个选中执行人**参与的条目
4. 按钮上出现蓝色数字徽标，表示当前选中数量
5. 点击面板内「清除筛选」取消所有选择

### 9.2 按配合团队筛选

1. 点击「**配合团队 ▼**」按钮，弹出团队列表（从所有条目的 `collaborators` 字段自动收集）
2. 勾选一个或多个团队名称
3. Roadmap 立即只显示**至少包含一个选中团队**的条目
4. 同样支持多选和清除

### 9.3 组合使用

两个筛选器同时生效时取**交集**：条目必须同时满足执行人条件和团队条件才会显示。

> **提示：** 筛选不影响数据本身，切换视图或刷新页面后筛选状态会重置。

## 13. 数据持久化与团队协作

### 云端存储

- 所有 Roadmap 数据存储在**云端**（Vercel Blob Storage），多人共享同一份数据
- 任何人编辑后，其他人**刷新页面**即可看到最新内容
- 换浏览器、换设备均可访问同一份数据，无需手动同步
- 工具栏右侧显示当前版本号和时间，例如：`v12 · 04-10 14:32`

### 冲突保护

当两人几乎同时编辑时，系统会自动检测版本冲突：
- 如果你的修改与他人冲突，会弹出提示：「数据已被他人更新，是否刷新页面获取最新数据？」
- 点击「确定」刷新获取最新版本；点击「取消」可先导出 CSV 备份你的修改

### CSV 备份

虽然数据已存储在云端，仍建议定期导出 CSV 作为备份：
1. 点击「**保存**」→ 得到 `roadmap_FY2026_v12.csv`
2. CSV 文件可用于备份恢复、离线查看、或交给 LLM 处理

### 本地 UI 偏好

以下设置保存在浏览器本地，不会同步到其他设备：
- 分组折叠/展开状态
- 左侧列宽设置

## 14. CSV 数据格式参考

完整示例：

```csv
author,module,pillar,problem,title,description,outcome,collaborators,startMonth,duration
Lynn,健康管理,用户留存,用户缺乏科学减重方案,Routines 减重计划,设计基于目标体重的阶段性计划,30天留存提升25%,医学团队;内容团队,4,3
Ned,健康管理,用户留存,通用食谱不满足个体差异,营养餐食推荐引擎,基于健康档案构建个性化食谱推荐,推荐满意度≥4.2分,医学团队;数据团队,7,2
Lynn,本地活动,用户增长,缺乏线下互动,文莱QR Code寻宝,布置实体QR码任务点,活动期间DAU提升40%,Ops Team;市场团队,5,2
```

**`startMonth` 对照表：**

| 值 | 月份 | 财年列位置 |
|----|------|-----------|
| 4 | 四月 | 第 1 列 |
| 5 | 五月 | 第 2 列 |
| 6 | 六月 | 第 3 列 |
| 7 | 七月 | 第 4 列 |
| 8 | 八月 | 第 5 列 |
| 9 | 九月 | 第 6 列 |
| 10 | 十月 | 第 7 列 |
| 11 | 十一月 | 第 8 列 |
| 12 | 十二月 | 第 9 列 |
| 1 | 一月（次年）| 第 10 列 |
| 2 | 二月（次年）| 第 11 列 |
| 3 | 三月（次年）| 第 12 列 |

**duration 上限：** 受财年约束，例如 11月开始最多 5 个月（到次年 3月），4月开始最多 12 个月。

## 15. 搭配 LLM 使用

推荐工作流：

```
1. 整理产品规划 → 让 LLM 输出标准 CSV
2. 导入 CSV → 可视化呈现
3. 在页面拖拽微调时间安排 / 重命名合并模块
4. 导出 CSV → 交给 LLM 继续优化
5. 重复以上循环
```

**给 LLM 的 Prompt 模板：**

```
请将以下产品规划整理成 Roadmap CSV 格式，严格遵守以下表头：

author,module,pillar,problem,title,description,outcome,collaborators,startMonth,duration

字段说明：
- author：负责人姓名（多人用英文逗号分隔）
- module：所属模块（相同名称会归为一组）
- pillar：所属高层价值主题（如「用户增长」「平台稳定性」，一个 Pillar 可跨多个模块；不确定时留空）
- problem：当前痛点或背景
- title：功能/项目名称（显示在时间轴条形上）
- description：详细描述
- outcome：完成后的价值或指标
- collaborators：协作团队，多个用英文分号分隔
- startMonth：开始月份，1-12 的整数（1=一月，4=四月）
- duration：持续月数，整数

要求：
- 只输出 CSV，第一行为表头，不要有多余文字
- startMonth 使用自然月数字
- pillar / collaborators 为空时留空即可

以下是产品规划内容：
[粘贴你的规划内容]
```
