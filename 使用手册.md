## 1. 打开应用

直接在 Finder 中双击 `index.html`，或通过终端启动本地服务器：

```bash
# 方式一：Python（推荐，无需安装）
python3 -m http.server 3000
# 然后访问 http://localhost:3000

# 方式二：npx serve
npx serve -p 3000 .
```

> **注意：** 直接双击 `file://` 打开与通过 `http://localhost` 打开均可正常使用，数据都存在浏览器 localStorage 中。

```
┌─────────────────────────────────────────────────────────────┐
│  EVYD Roadmap  ‹ 2026 ›   导入 JSON   导出   示例   + 添加  │  ← 顶部工具栏
├──────────┬─────┬─────┬─────┬─────┬─────┬─────┬─────────────┤
```

**时间轴说明：**

- 当前月份在表头高亮显示

## 3. 导入数据

### 3.1 导入 JSON 文件

1. 点击工具栏「**导入 JSON**」按钮
2. 选择本地 `.json` 文件
3. 数据立即渲染，并自动保存到本地

### 3.2 下载示例文件

点击工具栏「**示例**」按钮，下载包含 8 条示例数据的 JSON 文件，可作为导入格式参考。

### 3.3 格式要求

文件必须是合法 JSON，包含 `items` 数组：

```json
{
  "year": 2026,
  "items": [
    {
      "module": "模块名称",
      "title": "功能标题",
      "problem": "解决的问题（可选）",
      "outcome": "预期效果（可选）",
      "startMonth": 4,
      "duration": 3
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `module` | string | ✅ | 所属模块，相同名称自动归组 |
| `title` | string | ✅ | 条目标题，显示在条形上 |
| `problem` | string | ❌ | 解决的问题，显示在 tooltip |
| `outcome` | string | ❌ | 预期效果，显示在 tooltip |
| `startMonth` | number | ✅ | 开始月份，自然月 1–12（1=一月，4=四月） |
| `duration` | number | ✅ | 持续月数，1–12 |
| `id` | string | ❌ | 唯一标识，不填时自动生成 |

## 4. 手动添加条目

1. 点击工具栏「**+ 添加条目**」按钮（或按 `⌘N` / `Ctrl+N`）
2. 在弹出的表单中填写：
   - **所属模块**（必填）：直接输入模块名，相同名称自动合并到同一行
   - **功能/项目标题**（必填）：显示在条形上的名称
   - **解决的问题**（选填）：在 tooltip 中显示
   - **预期效果**（选填）：在 tooltip 中显示
   - **开始月份**：1–12，对应自然月
   - **持续时长**：月数
3. 表单底部的「预览」区域实时展示条形位置
4. 点击「**保存**」

> **提示：** 模块名称相同的条目会自动归到同一模块组，颜色也会一致。

## 5. 编辑与删除条目

**编辑：**
1. 单击任意条形，弹出编辑表单
2. 修改内容后点击「**保存**」

**删除：**
1. 单击条形，打开编辑表单
2. 点击左下角红色「**删除条目**」按钮
3. 在确认弹窗中点击「确认」

> **注意：** 删除操作不可撤销，如需找回请从导出的 JSON 文件恢复。

## 6. 拖拽调整

### 6.1 移动条目（调整开始月份）

- 鼠标**按住条形中间**区域并水平拖拽
- 松开后自动吸附到最近的月份
- 拖拽过程中条形半透明抬起，松手后保存

### 6.2 调整时长

- 鼠标**按住条形右侧深色手柄**（竖线区域）并水平拖拽
- 向右拖拽延长时长，向左缩短，最短 1 个月
- 条形宽度实时更新，徽标数字同步变化

> **注意：** 拖拽后松开鼠标才算保存，期间刷新页面会丢失未保存的拖拽。

---

## 7. 查看条目详情

**悬停 tooltip：**
- 鼠标悬停在条形上，自动弹出详情卡片
- 显示：模块名、时间范围、标题、解决的问题、预期效果
- 移开鼠标后自动隐藏

---

## 8. 折叠模块

- 点击左侧**模块名称行**，该模块下所有条目折叠/展开
- 折叠状态跨刷新保留（存储于 localStorage）
- 折叠后模块行右侧显示条目数量徽标

---

## 9. 切换年份

- 点击顶部年份左右箭头「**‹ 2026 ›**」切换
- 切换后展示对应财年（例：2026 = 2026年4月 → 2027年3月）
- 当前年份有今日指示线，其他年份无

## 10. 导出数据

1. 点击工具栏「**导出**」按钮
2. 浏览器自动下载 `roadmap_2026_v12.json` 文件（文件名含版本号）
3. 导出格式与导入格式完全一致，可直接再次导入或交给 LLM 修改

---

## 11. 数据持久化与跨浏览器同步

- 所有数据存储在浏览器 **localStorage**，键名为 `evyd_roadmap_data`
- 刷新页面、关闭再打开：数据**不会丢失**
- 清除浏览器缓存 / 换浏览器：数据会丢失，请先「导出」备份
- 不同浏览器、不同设备之间数据**不共享**

### 跨浏览器同步方式（JSON 即数据库）

**工具栏右侧**显示当前版本号和时间，例如：`v12 · 03-27 14:32`

**团队协作流程：**
1. 任何人完成修改后，点击「**导出**」→ 得到 `roadmap_2026_v12.json`
2. 将文件发给其他成员（微信、邮件、共享文件夹均可）
3. 其他成员点击「**导入 JSON**」，选择收到的文件
4. 若导入版本比本地旧，会弹出冲突警告，防止误覆盖

**建议：** 导出时文件名已含版本号，不要重命名，便于版本对比。

## 12. JSON 数据格式参考

完整示例：

```json
{
  "year": 2026,
  "items": [
    {
      "id": "i_abc123",
      "module": "数据展示",
      "title": "图表组件重构",
      "problem": "现有图表加载慢，交互卡顿，不支持大数据量",
      "outcome": "加载速度提升 50%，支持百万级数据点渲染",
      "startMonth": 4,
      "duration": 3
    },
    {
      "module": "用户增长",
      "title": "注册转化漏斗优化",
      "problem": "注册流程步骤繁琐，用户中途流失率高达 40%",
      "outcome": "注册转化率提升 20%，完成率 ≥ 75%",
      "startMonth": 6,
      "duration": 2
    }
  ]
}
```

**`startMonth` 对照表：**

| 值 | 月份 | 财年列位置 |
|----|------|-----------|
| 4 | 四月 | 第 1 列 |
| 5 | 五月 | 第 2 列 |
| 6 | 六月 | 第 3 列 |
| 7 | 七月 | 第 4 列 |
| 8 | 八月 | 第 5 列 |
| 9 | 九月 | 第 6 列 |
| 10 | 十月 | 第 7 列 |
| 11 | 十一月 | 第 8 列 |
| 12 | 十二月 | 第 9 列 |
| 1 | 一月（次年）| 第 10 列 |
| 2 | 二月（次年）| 第 11 列 |
| 3 | 三月（次年）| 第 12 列 |

## 13. 搭配 LLM 使用

推荐工作流：

```
1. 整理产品规划 → 让 LLM 输出标准 JSON
2. 导入 JSON → 可视化呈现
3. 在页面拖拽微调时间安排
4. 导出 JSON → 交给 LLM 继续优化
5. 重复以上循环
```

**给 LLM 的 Prompt 模板：**

```
请将以下产品规划整理成 Roadmap JSON 格式，严格遵守以下 Schema：

{
  "year": <年份，数字>,
  "items": [
    {
      "module": "<所属模块>",
      "title": "<功能/项目名称>",
      "problem": "<当前痛点或背景>",
      "outcome": "<完成后的价值或指标>",
      "startMonth": <开始月份，1-12的整数，1代表一月>,
      "duration": <持续月数，1-12的整数>
    }
  ]
}

要求：
- module 相同的条目会归为一组，请合理归类
- startMonth 使用自然月（4月 = 4，1月 = 1）
- 只输出 JSON，不要有多余文字

以下是产品规划内容：
[粘贴你的规划内容]
```
