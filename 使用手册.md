## 1. 打开应用

直接在 Finder 中双击 `index.html`，或通过终端启动本地服务器：

```bash
# 方式一：Python（推荐，无需安装）
python3 -m http.server 3000
# 然后访问 http://localhost:3000

# 方式二：npx serve
npx serve -p 3000 .
```

> **注意：** 直接双击 `file://` 打开与通过 `http://localhost` 打开均可正常使用，数据都存在浏览器 localStorage 中。

## 3. 导入数据

### 3.1 导入 CSV 文件

1. 点击工具栏「**加载 CSV**」按钮
2. 选择本地 `.csv` 文件（支持 Excel 导出的 UTF-8 CSV）
3. 数据立即渲染，并自动保存到本地
4. **导入会替换全部现有数据**，请先「保存」备份

### 3.2 下载示例文件

点击工具栏「**示例**」按钮，下载包含示例数据的 CSV 文件，可用 Excel 或文本编辑器查看格式。

### 3.3 格式要求

首行为表头，后续每行一条条目：

```csv
author,module,problem,title,description,outcome,collaborators,startMonth,duration
Lynn,数据展示,现有图表加载慢,图表组件重构,重构图表渲染层,加载速度提升50%,,4,3
Ned,用户增长,注册流失率高,注册漏斗优化,简化注册步骤,转化率提升20%,市场团队,6,2
```

| 列 | 必填 | 说明 |
|----|------|------|
| `author` | ❌ | 负责人姓名，显示在条形左侧头像 |
| `module` | ✅ | 所属模块，相同名称自动归组 |
| `problem` | ❌ | 解决的问题，显示在 tooltip |
| `title` | ✅ | 条目标题，显示在条形上 |
| `description` | ❌ | 详细描述，显示在 tooltip |
| `outcome` | ❌ | 预期效果，显示在 tooltip |
| `collaborators` | ❌ | 协作团队，多个用 `;` 分隔 |
| `startMonth` | ✅ | 开始月份 1–12（1=一月，4=四月） |
| `duration` | ✅ | 持续月数，最长延伸至财年末（次年3月） |

## 4. 手动添加条目

1. 点击工具栏「**+ 添加条目**」按钮（或按 `⌘N` / `Ctrl+N`）
2. 在弹出的表单中填写：
   - **所属模块**（必填）：直接输入模块名，相同名称自动合并到同一行
   - **功能/项目标题**（必填）：显示在条形上的名称
   - **负责人**（选填）：显示在条形左侧
   - **协作团队**（选填）：多个用 `;` 分隔
   - **解决的问题** / **预期效果** / **详细描述**（均选填）
   - **开始月份**：1–12，对应自然月
   - **持续时长**：月数
3. 表单底部的「预览」区域实时展示条形位置
4. 点击「**保存**」

> **提示：** 模块名称相同的条目会自动归到同一模块组，颜色也会一致。

## 5. 编辑与删除条目

**编辑：**
1. 单击任意条形，弹出编辑表单
2. 修改内容后点击「**保存**」

**删除：**
1. 单击条形，打开编辑表单
2. 点击左下角红色「**删除条目**」按钮
3. 在确认弹窗中点击「确认」

> **注意：** 删除操作不可撤销，如需找回请从导出的 CSV 文件恢复。

---

## 6. 拖拽调整

### 6.1 移动条目（调整开始月份）

- 鼠标**按住条形中间**区域并水平拖拽
- 松开后自动吸附到最近的月份
- 拖拽过程中条形半透明抬起，松手后保存

### 6.2 调整时长

- 鼠标**按住条形右侧深色手柄**（竖线区域）并水平拖拽
- 向右拖拽延长时长，向左缩短，最短 1 个月
- 时长可以**跨越日历年**，最长延伸到财年末（次年 3 月）
- 条形宽度实时更新，徽标数字同步变化

> **注意：** 拖拽后松开鼠标才算保存，期间刷新页面会丢失未保存的拖拽。

---

## 7. 模块管理

### 7.1 重命名模块

- **双击**左侧模块名称，进入编辑状态
- 修改名称后按 `Enter` 确认，或按 `Esc` 取消
- 该模块下所有条目的 `module` 字段同步更新

### 7.2 合并模块

- 将某个模块重命名为另一个已存在的模块名
- 两组条目自动合并到同一模块

### 7.3 拖拽排序

- 鼠标**按住模块左侧拖拽手柄**（⠿ 图标）并上下拖拽
- 松开后模块顺序即时更新并保存

## 8. 查看条目详情

**悬停 tooltip：**
- 鼠标悬停在条形上，自动弹出详情卡片
- 显示：负责人、模块名、时间范围、标题、问题、描述、效果、协作团队
- 移开鼠标后自动隐藏

---

## 9. 折叠模块

- 点击左侧**模块名称行**，该模块下所有条目折叠/展开
- 折叠状态跨刷新保留（存储于 localStorage）
- 折叠后模块行右侧显示条目数量徽标

## 10. 导出数据

1. 点击工具栏「**保存**」按钮
2. 浏览器自动下载 `roadmap_FY2026_v12.csv` 文件（文件名含财年和版本号）
3. 导出格式与导入格式完全一致，可用 Excel 查看，也可直接再次导入或交给 LLM 修改

## 11. 数据持久化与团队协作

- 所有数据存储在浏览器 **localStorage**，键名为 `evyd_roadmap_data`
- 刷新页面、关闭再打开：数据**不会丢失**
- 清除浏览器缓存 / 换浏览器：数据会丢失，请先「保存」备份
- 不同浏览器、不同设备之间数据**不共享**

### 跨设备同步方式（CSV 即数据库）

**工具栏右侧**显示当前版本号和时间，例如：`v12 · 03-27 14:32`

**团队协作流程：**
1. 任何人完成修改后，点击「**保存**」→ 得到 `roadmap_FY2026_v12.csv`
2. 将文件发给其他成员（微信、邮件、共享文件夹均可）
3. 其他成员点击「**加载 CSV**」，选择收到的文件
4. 若导入版本比本地旧，会弹出冲突警告，防止误覆盖

**建议：** 导出时文件名已含版本号，不要重命名，便于版本对比。

## 12. CSV 数据格式参考

完整示例：

```csv
author,module,problem,title,description,outcome,collaborators,startMonth,duration
Lynn,健康管理,用户缺乏科学减重方案,Routines 减重计划,设计基于目标体重的阶段性计划,30天留存提升25%,医学团队;内容团队,4,3
Ned,健康管理,通用食谱不满足个体差异,营养餐食推荐引擎,基于健康档案构建个性化食谱推荐,推荐满意度≥4.2分,医学团队;数据团队,7,2
Lynn,本地活动,缺乏线下互动,文莱QR Code寻宝,布置实体QR码任务点,活动期间DAU提升40%,Ops Team;市场团队,5,2
```

**`startMonth` 对照表：**

| 值 | 月份 | 财年列位置 |
|----|------|-----------|
| 4 | 四月 | 第 1 列 |
| 5 | 五月 | 第 2 列 |
| 6 | 六月 | 第 3 列 |
| 7 | 七月 | 第 4 列 |
| 8 | 八月 | 第 5 列 |
| 9 | 九月 | 第 6 列 |
| 10 | 十月 | 第 7 列 |
| 11 | 十一月 | 第 8 列 |
| 12 | 十二月 | 第 9 列 |
| 1 | 一月（次年）| 第 10 列 |
| 2 | 二月（次年）| 第 11 列 |
| 3 | 三月（次年）| 第 12 列 |

**duration 上限：** 受财年约束，例如 11月开始最多 5 个月（到次年 3月），4月开始最多 12 个月。

## 13. 搭配 LLM 使用

推荐工作流：

```
1. 整理产品规划 → 让 LLM 输出标准 CSV
2. 导入 CSV → 可视化呈现
3. 在页面拖拽微调时间安排 / 重命名合并模块
4. 导出 CSV → 交给 LLM 继续优化
5. 重复以上循环
```

**给 LLM 的 Prompt 模板：**

```
请将以下产品规划整理成 Roadmap CSV 格式，严格遵守以下表头：

author,module,problem,title,description,outcome,collaborators,startMonth,duration

字段说明：
- author：负责人姓名
- module：所属模块（相同名称会归为一组）
- problem：当前痛点或背景
- title：功能/项目名称（显示在时间轴条形上）
- description：详细描述
- outcome：完成后的价值或指标
- collaborators：协作团队，多个用英文分号分隔
- startMonth：开始月份，1-12 的整数（1=一月，4=四月）
- duration：持续月数，整数

要求：
- 只输出 CSV，第一行为表头，不要有多余文字
- startMonth 使用自然月数字
- collaborators 为空时留空即可

以下是产品规划内容：
[粘贴你的规划内容]
```
